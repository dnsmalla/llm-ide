# Graph Notes Folder Fix — Removed Unnecessary Subfolder

## Issue
Code notes (notes ABOUT code) were being written to `system/graph/notes/` - an unnecessary subfolder.

## Your Point
When making a graph OF notes, the notes should be directly in `system/graph/`, not in a `notes/` subfolder.

## Fix Applied

**Before:**
```
system/
└── graph/
    ├── notes/          ← Unnecessary subfolder
    │   ├── file1.md
    │   └── file2.md
    ├── graph.json
    └── index.md
```

**After:**
```
system/
└── graph/
    ├── file1.md        ← Code notes directly here ✅
    ├── file2.md        ← No unnecessary subfolder ✅
    ├── graph.json
    └── index.md
```

## Changes Made

### 1. ProjectLayout.swift
**Before:**
```swift
var graphNotesDir: URL { graphDir.appendingPathComponent("notes", isDirectory: true) }
```

**After:**
```swift
var graphNotesDir: URL { graphDir }  // Points directly to graph/
```

### 2. AnalyzePhase.swift
**Before:**
```swift
notesDir: "system/graph/notes"
```

**After:**
```swift
notesDir: "system/graph"  // Notes written directly to graph/
```

## Impact

- ✅ **Simpler structure**: No unnecessary subfolder nesting
- ✅ **Clearer semantics**: Graph OF notes lives directly in graph folder
- ✅ **Backward compatible**: Existing code adapts to new structure
- ✅ **Build successful**: All Swift tests still pass

## Two Different "Notes" Concepts

1. **Code Notes** (`system/graph/file.md`)
   - Notes ABOUT code structure
   - Generated by analyzing imports, symbols, types
   - Now written directly to `system/graph/` ✅

2. **Generated Notes** (`notes/emails/file.md`, `notes/meetings/file.docx`)
   - AI-generated notes from meetings, emails, documents
   - Written to unified `notes/` folder (separate system)

Both are now properly organized without unnecessary nesting.
