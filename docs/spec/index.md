# Spec — rebuild-grade contracts

This is the **specification layer**: the exact contracts a reader needs to *reproduce* a subsystem, not just understand it. It complements [`../explanation/`](../explanation/architecture.md) — explanation tells you *what and why*; spec tells you *exactly what*.

**Intended reader:** a fresh engineer or AI with only these spec pages plus the structured reference pages they link (e.g. [`../reference/database-schema.md`](../reference/database-schema.md), [`../reference/api/openapi.yaml`](../reference/api/openapi.yaml)). They should be able to rebuild the subsystem without inventing detail.

## The "generative-grade" bar

A spec page passes when:

- Every symbol / endpoint / table / prompt it governs is present with its **exact** shape — no "etc.", no "see code".
- Every magic number, timeout, cap, regex, and crypto parameter is stated.
- A spot-check rebuild of one representative piece (recreate a table's DDL, or one endpoint's request/response) from the page alone matches the source.
- Structured facts **link to their extractor-generated reference page** rather than being hand-copied (so they can't drift).

## Regeneration checklist

Every spec page ends with this checklist:

```markdown
## Regeneration checklist
- [ ] Every governed symbol/endpoint/table/prompt is present with its exact shape (no "etc.", no "see code").
- [ ] Every magic number, timeout, cap, regex, and crypto parameter is stated.
- [ ] Spot-check: one representative piece rebuilt from this page alone matches source.
- [ ] Structured facts link to their extractor-generated reference page (no hand-copied drift).
```

## Staying in sync

Structured facts are generated by `docs/_scripts/extract_*.py` and guarded by `docs/_scripts/test_*.py` plus `docs/_scripts/check_api_coverage.py`. When you change a contract, regenerate the reference page and run the extractor tests — never hand-edit a generated page.
