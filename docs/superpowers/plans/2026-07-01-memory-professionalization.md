# Memory system — review & professionalization roadmap

_2026-07-01. Status: Tier 1 shipped; Tiers 2–3 proposed._

A review of how repo memory is generated and used across the Mac app and the
Node backend, and a prioritized plan to bring it to professional grade. Scope is
the "Graphify" repo memory the Code Assistant grounds answers in — **not** the
per-session chat transcript (which is local-only and never sent to the server).

## How memory is generated & used

Four artifacts live under `<repo>/graphify-out/memory/` and are injected into the
agent's persona base on **every** `/code-assist` turn via
`renderGraphifyMemory()` (`extension/graphkit/memory.mjs`):

| Artifact | Generated by | When | Injection priority |
|---|---|---|---|
| `chat-memory.md` — curated durable facts | LLM extraction (`llm_agent/runtime/memory-extract.mjs`), ≤5 facts/turn, deduped, written by `graphkit/memory-writer.mjs` | after every turn (fire-and-forget, `memory-persist.mjs`) | 2 |
| `repo.md` — impact-ranked repo overview | `KnowledgeGraphService.writeMemoryArtifact` (Swift) | `GraphAutoUpdater`: on project open, a 15-min timer, and FSEvents (2 s debounce) | 1 |
| `graph-notes.md` / `doc-notes.md` | same | same | 3 / 4 |

- **Read/budget** (`memory.mjs`): each present repo's block is bounded — 4 KB per
  file, **8 KB for `chat-memory.md`** (matches the write cap), 16 KB total, 2 repos.
  Highest-signal content is added first so it survives a tight budget, and
  truncation is line-boundary-safe (`clipToBoundary`) so a fact is never split
  mid-sentence.
- **Security**: every read and write passes `resolveAllowedRepoRoot()` — tenancy
  allow-list + `..`/relative-path rejection, APFS case-normalized. Reader and
  writer share the gate.
- **Freshness**: the doc set is fingerprinted (`path|size|mtime` SHA-256) so an
  unchanged re-generation is near-free; block headers carry an `(updated ~N ago)`
  clause, and an allow-listed repo with no memory emits an explicit
  "not generated yet" marker rather than contributing nothing.
- **UI**: `ProjectMemoryView` shows/edits `chat-memory.md` facts;
  `GraphMemorySettingsSection` shows all four artifacts read-only with mtimes.

## Assessment

Strong foundations: allow-list gate on every access, doc-set fingerprinting,
run-coalescing in the auto-updater, dedup, and a prioritized + boundary-safe
budget. The gaps are in observability, tunability, fact quality, and data
lifecycle.

## Roadmap

### Tier 1 — correctness (shipped)
- **Prioritized + boundary-safe budget** — overview + curated facts survive a
  tight budget; truncation cuts on line boundaries. _(commit `495b4ea`)_
- **Full chat-memory injection** — the reader clipped `chat-memory.md` to 4 KB
  while the writer stores 8 KB, so half the curated facts never reached the
  agent. Now read at the full 8 KB cap, with a regression test. _(commit `ee0d73c`)_
- **Verified non-issue:** the extraction model can be `undefined`, but
  `runClaude` falls back to its default and the call is try/caught — it degrades
  gracefully; no hardcoded model default was added (it would be worse).

### Tier 2 — observability & control
1. **Injection observability (shipped)** — the `memory_context` log now carries a
   per-file breakdown (`files: [{ repo, file, chars, truncated }]`, `truncatedAny`)
   of what actually reached the agent, via an optional `stats` sink on
   `renderGraphifyMemory`. _(commit `d1bc28a`)_
2. **Eviction visibility (shipped)** — `appendChatMemory` takes an optional `meta`
   sink reporting `{ evicted, added }`; `memory-persist` folds `evicted` into the
   `project_memory` log so hitting the 100-fact / 8 KB cap is no longer silent.
3. **Configurable limits (backend shipped)** — the memory budgets now live in
   `config.memory` (`core/config.mjs`), env-tunable: `LLMIDE_MEM_PER_FILE_CHARS`,
   `LLMIDE_MEM_TOTAL_CHARS`, `LLMIDE_MEM_MAX_REPOS`, `LLMIDE_MEM_CHAT_FILE_CHARS`
   (shared reader+writer cap — can't drift), `LLMIDE_MEM_MAX_FACTS`; documented in
   `.env.example`. The Mac-side auto-update interval is now an `AppConfig`
   setting (`graphAutoUpdateMinutes`, default 15, floor 5) with a Stepper in the
   Settings → Graph & Memory card; `GraphAutoUpdater.setIntervalMinutes`
   reschedules the timer live.
4. **Extraction cost tracking (shipped)** — `extractMemories` reports an
   `approxTokens` estimate via an optional `meta` sink; `memory-persist` logs it
   as `extractTokens` on both the `no_facts` and `captured` outcomes (the call
   is paid either way). New `LLMIDE_MEM_*` vars are documented in `.env.example`.

**Tier 2 is complete.**

### Tier 3 — fact quality & lifecycle
5. **Structured facts — category shipped.** Extraction now emits a category per
   fact (convention / architecture / tooling / command / preference), stored
   inline as `[category] fact`; dedup keys on the fact text (strips the tag) so a
   fact can't re-enter under a different category, and the reader/UI need no
   change (the tag is human- and agent-readable). Legacy untagged facts still
   parse. **Remaining:** per-fact timestamp + source turn (needs the writer to
   preserve per-fact metadata across rewrites), and category-aware pruning.
6. **UI parity** — let users see/curate all four memory kinds the agent receives,
   not just chat facts.
7. **Versioning / export** — snapshot or audit trail so evicted/deleted facts are
   recoverable; structured (JSON) export.
8. **De-dup shared helpers** — `factKey`/`relativeAge` are duplicated across
   modules; `graph-notes` cross-links truncate at 50 silently.

## Key files
- `extension/graphkit/memory.mjs` — read/assemble/budget the memory block.
- `extension/graphkit/memory-writer.mjs` — write/dedup/cap `chat-memory.md`.
- `extension/llm_agent/runtime/memory-extract.mjs` — LLM fact extraction.
- `extension/llm_agent/runtime/memory-persist.mjs` — per-turn capture + logging.
- `mac/Sources/LlmIdeMac/CodeGraph/KnowledgeGraphService.swift` — graph artifacts.
- `mac/Sources/LlmIdeMac/CodeGraph/GraphAutoUpdater.swift` — regen cadence.
- Tests: `extension/tests/project-memory.test.mjs`, `graphify-memory-*.test.mjs`.
